# 与原100档版本对比

本文档记录当前开发版相对 **【原100档版本】** 的差异，以及 `modinfo` 中 `roge` 配置项的实际作用范围。

## 术语

| 名称 | 对应路径 / 标识 | 版本 |
|------|-----------------|------|
| **原100档版本** | `depot_322330_6793828808048834807/` | `4.9.8.1` |
| **当前开发版** | 仓库根目录（排除 `depot_*` 目录） | 见 `modinfo.lua` 中 `version`（如 `6.8.5`） |

另：`depot_322330_6606165671023863466/` 为 Steam 上较新的 `4.9.8.2` 快照，相对原100档版本仅差版本号与 `postinit/jiaqiang.lua` 中潜伏梦魇（`shadowdragon`）可作祟补丁，与当前开发版无直接对照关系。

## `roge` 配置项

`modinfo.lua` 中的配置：

```lua
{ name = 'roge', label = zh_en('是否启用骰子肉鸽模式(roge模块)', 'Enable dice roguelike module (roge folder)'), ... default = true }
```

`modmain.lua` 中唯一通过 `load_if_enabled("roge", ...)` 加载的入口：

```lua
load_if_enabled("roge", "postinit/roge/init.lua")
```

全仓库内 **`GetModConfigData("roge")` 仅出现在两处**：

1. `modmain.lua` — 是否加载 `postinit/roge/init.lua`
2. `postinit/spat.lua` — 是否应用钢羊 roge 数值加强（移速 / 攻击间隔）

`postinit/roge/` 内部子模块**没有**二次 `GetModConfigData("roge")` 检查。

## 差异分类

### 关闭 `roge` 可以禁用的内容

`roge = No` 时不会加载 `postinit/roge/init.lua`，从而跳过以下子模块：

| 模块文件 | 功能 |
|----------|------|
| `postinit/roge/roll.lua` | 骰子规则、Shift+T/Y/U 召唤、天数解锁 |
| `postinit/roge/boss_pool.lua` | 怪物池、Boss 血量、各类 roge 召唤 |
| `postinit/roge/dice_hud_buttons.lua` | 韦伯 HUD 骰子按钮 |
| `postinit/roge/ghost.lua` | 作祟/附身天数限制、韦伯鬼魂逻辑 |
| `postinit/roge/daywalker2_roge.lua` | roge 拾荒疯猪 Boss 包 |
| `postinit/roge/walrus_squad.lua` 等 | 海象 / 鱼人 / 猪人精英召唤包 |
| `postinit/roge/recipes.lua` | 强心针 / 恐惧弹丸配方改动 |
| `postinit/roge/starting_supplies.lua` | 开局物资 |
| `postinit/roge/wickerbottom.lua` | roge 下薇克巴顿改动 |
| `postinit/roge/void_protection.lua` | 洞穴虚空保护 |
| `postinit/spat.lua`（部分） | 仅 roge 数值加强；附身吐痰修复仍会加载 |

### 文件名含 roge、但未接入 `roge` 开关

以下文件在 `roge = No` 时**仍会生效**：

| 文件 | 说明 |
|------|------|
| `postinit/roge_portal_pull_immunity.lua` | `modmain` 无条件 `modimport`；处理 `roge_portal_rift` 吸入免疫 |
| `postinit/daywalker2_laser_splash.lua` | 无条件加载；`RogeDaywalker2LaserSplashCapture/Restore()` 始终执行 |
| `postinit/wortox.lua` | 无条件加载；含 `_roge_wortox_*` 灵魂治疗、灵魂跳跃改动 |
| `postinit/wolfgang.lua` | 在 `wolfgang` 配置开启时加载；含 `_roge_no_heal_bonus_*` 逻辑 |
| `postinit/bishop.lua` | 无条件加载；注册 `_roge_special_attacks["bishop"]` |

### 与 roge 无关、无法用 `roge` 还原原100档版本的内容

#### `modmain.lua` 新增且无条件加载

- `postinit/ancient_altar_protect.lua` — 远古科技塔不可摧毁

## 结论

**关闭 `roge` 不能使当前开发版等价于原100档版本。**

`roge = No` 仅禁用「骰子肉鸽核心模块（`postinit/roge/*`）+ 钢羊 roge 数值加强」。关闭后仍与原100档版本存在显著差异，主要原因：

1. **Bishop / 梦魇体系**（`bishop.lua`、`scripts/nightmare/*`、`modworldgenmain`、眼球塔 SG）始终加载
2. **roge 命名但未 gated** 的 portal 免疫、daywalker2 激光、wortox / wolfgang 改动
3. **当前开发版相对 4.9.8.1 的通用加强**（人物、`jiaqiang`、prefab 等）走各自 config，与 `roge` 无关
4. **wigfrid 强制启用**、lobby UI、远古塔保护等原100档版本没有的功能

若目标是「`roge = No` 时尽量接近原100档版本」，需将上文「未接入 roge 开关」及「与 roge 无关的新增加载」进一步拆分或挂到 `roge` / 独立 config 下。

## 后续 gating 建议（参考）

若实现「`roge = No` ≈ 原100档版本 + 其它 config 保持默认」，可考虑在 `roge = No` 时额外跳过：

```
├── postinit/roge_portal_pull_immunity.lua
├── daywalker2_laser_splash + Capture/Restore
├── scripts/nightmare/*（若确认仅为 roge 服务）
├── postinit/bishop.lua / bishop_spawner / SGbishop
├── modworldgenmain 中 roge 相关 spawner 密度
├── postinit/lobby_changes.lua（若确认 roge 专用）
└── wortox / wolfgang 中 _roge_* 段落改为 if GetModConfigData("roge")
```

建议保留为独立 config、不随 `roge` 关闭的内容：

- `ancient_altar_protect`、`spider_robot_possess` 附身修复
- 各人物 / Boss 加强（现有 `boss` / 角色开关）
- `jiaqiang` 通用数值
